What is Functional Programming?

Functional Programming (FP) is a programming paradigm that treats computation as the evaluation of mathematical functions. Instead of using objects and changing their states (like OOP), FP focuses on using pure functions, avoiding shared state, and avoiding side effects.

üß† Key Principles of Functional Programming

1. Pure Functions

A pure function always gives the same output for the same input and does not change anything outside the function. It doesn‚Äôt use or modify global variables or states.

2. Immutability

In FP, data is not changed. Instead of modifying an array or object, you create a new one with the updated value.

3. First-Class and Higher-Order Functions

Functions are treated like data. You can pass them as arguments, return them from other functions, and assign them to variables.

4. No Side Effects

Side effects are changes outside the function like logging to the console, modifying a global variable, or reading a file. FP tries to avoid them to make programs easier to debug and test.

5. Function Composition

Small, simple functions are combined to build more complex ones. This makes the code modular and easier to manage.

üí° Why Learn FP?

  • Improves reliability and reduces bugs
  • Great for concurrency and multi-threading
  • Popular in data processing and functional languages (e.g., Haskell, Lisp)
  • Used widely in JavaScript, Python, Scala, etc.

üíª Example: Pure Function in JavaScript

// Pure Function
function add(a, b) {
  return a + b;
}

// Not pure (side effect: console.log)
function greet(name) {
  console.log("Hello " + name);
}

Higher-Order Function Example

// Passing a function as a parameter
function greet(name) {
  return "Hi, " + name;
}

function processUser(callback) {
  const result = callback("Wesley");
  console.log(result);
}

processUser(greet); // Output: Hi, Wesley
