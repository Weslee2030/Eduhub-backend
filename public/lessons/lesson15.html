üóÉÔ∏è What is a Database?

A database is an organized collection of data that can be easily accessed, managed, and updated. It's used to store information for websites, apps, and systems.

üìö SQL Databases (Relational)

Structured Query Language (SQL) databases store data in rows and columns inside tables. They enforce strict schemas and relationships between data.

  • Popular SQL databases: MySQL, PostgreSQL, SQLite, Microsoft SQL Server
  • Best for structured data with relationships
  • Uses SQL language for operations like:
SELECT * FROM users WHERE age > 18;
INSERT INTO users (name, email) VALUES ('John', 'john@example.com');

üå©Ô∏è NoSQL Databases (Non-relational)

NoSQL databases use flexible data models like documents, key-value pairs, or graphs. Great for unstructured or semi-structured data.

  • Popular NoSQL databases: MongoDB, Firebase, Redis, Cassandra
  • Best for scalability and fast development
  • Example (MongoDB):
// Find users older than 18
db.users.find({ age: { $gt: 18 } });

// Insert new user
db.users.insertOne({ name: "Alice", email: "alice@example.com" });

üîÅ SQL vs NoSQL Summary

  • SQL: Table-based, schema-defined, strong consistency
  • NoSQL: Document/key-value/graph-based, schema-less, high flexibility
